Enhancement 1 - Audio Player

I have used the inbuilt HTML Audio Player to allow users to listen to the tracks detailed on the music page.
The controls attribute adds audio controls, like play, pause, and volume.
The source element sets the location and the type of sudio file to be played.
One file had to be converted to .mp3 before it could be used as .wma file type is not supported. This is purely for showcase purposes, if this website was to be published the fileswould have to be removed or risk copy-right infringement.

Enhancement 2 - Responsiveness

All pages on this website are responsive ie. the elements change size relative to the size of the screen they are being viewed on.
This has been done by using relative values for much of the layout eg. 50% page width here

There are limits to this function though. On the index page the width cannot go below a certain width before the nav-bar starts to become hidden. The height of the page also cannot 50% without the nav bar disappearing out of view.
The other pages are more responsive. Most of the content move with the page width, the exception being the photos on the travel page which have fixed dimensions.
The website is not fully responsive/fluid as the side navigation bar has a fixed position and dimensions that do not change.
